## 这个工具的诞生源于我无法忍受 `react-router` 愚蠢的架构设计与实现。

相信许多开发者在使用时，也忍不住像我一样想问：


- 为什么 `navigate()` 这样一个 **全局性** 的函数，必须要在组件内通过 `useNavigate` 的方式才能够拿到？
- 为什么 URL 中的 query 参数这样 **全局性** 的信息，也必须在组件内通过 `useParams` 来拿到？
- 为什么会有 `<Navigate />` 这样奇怪的组件，通过渲染组件的方式来实现路由跳转？
- 为什么你们的一切设计都和 React Hooks 强绑定，而忽略了许多 API 根本不应该依赖视图组件的事实？



这并不是 `react-router` 独有的问题。事实上，**all in hooks** 的“流行”开源库比比皆是。  

使用 hooks 很酷，但问题在于：它是否真的应该用 hooks 来实现？  
如果在非 React 环境中，它还能正常工作吗？
